Flow Layout

Block elements take up the full horizontal space. Notice the heading above. The heading occupies all the available width even though it has only 11 characters. Most of the elements like div, h1, p, form, and so forth are block elements. They stack along the vertical direction.

Inline elements take up only space that is required. This span in red is inline. Inline elements like em, strong, span, and so forth grow in the horizontal direction and wrap to the next line if current line cannot fit the element.

Inline block elements act as block elements internally, while externally rendered as inline. The button tag and select tag are inline-block elements by default. The button has a width and a height but does not occupy the whole horizontal space. Also notice that, inline-block elements do not line-wrap.

The display property can be set to override the default behaviour. See the div boxes acting as inline elements below. Also notice that the left and right margins of the boxes do not collapse and the top and bottom margins are ignored. It is not recommended to set border, margin, and padding properties on inline elements as the behavior is not well-defined.
